## 📄 Overview

<a href="#lANScan">LANScan</a> is a network scanner and device discovery tool for Mac that reveals every device connected to the local network — displaying IP addresses, MAC addresses, hostnames, and device vendor information for every active device on the network with a simple, readable interface.

The <a href="#lANScan">fast network device discovery</a> scans the local subnet and identifies every active device — sending ping and ARP requests across the network address range and collecting responses to build a complete picture of which devices are online at the moment of the scan. Home networks with a mix of computers, phones, tablets, smart TVs, streaming devices, IoT sensors, and network equipment all appear in the scan results with their identifying information. The <a href="#lANScan">MAC address and vendor display</a> identifies the hardware manufacturer of each device from its MAC address — showing that a specific IP address belongs to an Apple device, a Samsung device, a Raspberry Pi, or any other manufacturer whose MAC address prefix is in the vendor database. This vendor identification immediately categorizes unfamiliar devices and helps identify unexpected network participants.

The <a href="#lANScan">hostname resolution</a> attempts to resolve the hostname for each discovered device — showing device names alongside IP and MAC addresses for devices that broadcast their names on the network. Computers and some smart devices broadcast recognizable hostnames that make them immediately identifiable in the scan results without requiring manual IP-to-device mapping. The <a href="#lANScan">open port scanning</a> checks which ports are open on discovered devices — identifying which services are running on network devices, verifying that expected services are accessible, and spotting unexpected open ports that might indicate security concerns or misconfiguration.</p>

<a href="#lANScan">Device history and comparison</a> tracks which devices have appeared on the network over time — comparing the current scan to previous scans to identify new devices that have joined the network since the last scan, useful for spotting unauthorized network access or tracking which guests have connected. <a href="#lANScan">Export to CSV and other formats</a> saves scan results for documentation, reporting, and integration with other network management workflows — recording the device inventory for network documentation or compliance purposes.

<a href="#lANScan">Scheduled scanning</a> runs network scans automatically at configured intervals — maintaining an up-to-date picture of network device inventory without manual scan initiation, and detecting when new devices appear or when known devices go offline. <a href="#lANScan">Simple and readable interface</a> presents the network scan results in a clean table that non-technical home users can understand alongside the technical detail that network administrators need — accessible to both audiences without requiring network expertise to interpret.</p>
